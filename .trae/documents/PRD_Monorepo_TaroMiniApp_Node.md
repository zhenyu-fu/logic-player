# PRD：logic-player Monorepo（Taro 微信小程序 + Node 后端）

## 目标
在一个 monorepo 内同时维护：
- 微信小程序端：用户自定义股票列表，并对每只股票记录买入逻辑 / 持有逻辑 / 卖出逻辑
- 后端服务：提供健康检查与基础数据 API（可逐步演进为账号、同步与多端数据一致性）

## 核心功能点
- 股票清单
  - 股票列表：新增/删除（本地持久化）、查看更新时间
  - 编辑股票：股票代码 + 名称
- 逻辑记录
  - 选择某只股票进入记录页
  - 编辑三段逻辑：买入逻辑 / 持有逻辑 / 卖出逻辑（本地持久化）
  - 记录列表：展示哪些股票已记录、最近更新时间
- 我的
  - 设置入口
  - 关于信息

## 页面列表（微信小程序）
| 页面名称 | 目录名 | 页面类型 | 说明 |
|---|---|---|---|
| 股票 | stocks | tabBar | 股票列表 + 新增入口 |
| 记录 | records | tabBar | 记录概览（已记录/未记录） |
| 我的 | mine | tabBar | 设置/关于 |
| 编辑股票 | stockEdit | 二级 | 新增/编辑股票 |
| 逻辑记录 | recordEdit | 二级 | 编辑买入/持有/卖出逻辑 |
| 设置 | settings | 二级 | 占位页 |

## API（后端）
- `GET /health`：健康检查
- `GET /api/stocks`：返回股票列表（当前为示例数据）
